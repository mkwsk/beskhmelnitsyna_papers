# Машиночитаемые пункты и ключи методик

Этот каталог нужен, чтобы карточки из `methods/*.md` можно было использовать в скриптах генерации форм и подсчета баллов.

## Политика хранения текста пунктов

Если правовой статус полного текста методики не подтвержден, публичный репозиторий хранит только структуру: номера пунктов, имена переменных, направление подсчета, принадлежность к шкалам и ссылку на источник. Полные формулировки можно добавить в приватной рабочей копии после проверки источника.

## Базовый формат `*_items.csv`

```csv
method_id,item_number,item_code,variable,text_a,text_b,text,scale_code,keyed_value,scoring_direction,required,source_note
```

- `text_a` и `text_b` используются для forced choice A/B.
- `text` используется для обычных утверждений Лайкерта.
- `keyed_value` хранит зачетный вариант для forced choice.
- `scoring_direction`: `direct`, `reverse`, `keyed`, `not_applicable`.
- Для многоключевых методик, где один пункт входит в несколько шкал, используется отдельный `*_key.csv`.

## Статусы подсчета

- `complete`: ключ достаточен для автоматического подсчета.
- `complete_key_without_item_texts`: ключ достаточен, но тексты пунктов не опубликованы в репозитории.
- `external_key_required`: перед автоматическим подсчетом нужно сверить ключ с первоисточником.
